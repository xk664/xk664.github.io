# 单列集合：Collection(接口)
	实现类：
		List：ArrayList，Linkedlist，Vector(有序，有重复，有索引)
		Set：HashSet，Treeset，LinkedHashSet(无序，无重复，无索引)

## 1.Arrays
***注：Arrays只能用于包装类***
#### Arrays.tostring(数组名):
			将数组拼接为字符串，以[1,2,3,4]的形式输出
### Arrays.sort()
**原理：二分加插入排序**
			先将下标为0的元素看作是有序序列，然后后面的元素一次插入
			默认为升序`public int compare(Integer o1, Integer o2)`o1为待排序元素，o2为有序元素中的中间值(二分)
			当o1-o2>=0时,会继续看后半段
			当o1-o2<0时，会在前半段
降序：
```Java
public .....implements comparator
Integer [] arr={4,1,3,5,4};  
  Arrays.sort(arr,new Comparator<Integer>(){  
      @Override  
      public int compare(Integer o1, Integer o2) {  
          return o2-o1;  
      }  //可用lambda代替
  });
```
升序：Arrays.sort(arr);
### Arrays.binarysearch(数组名，带查找元素)
**原理：二分查找**
**前提：序列为单调序列**
			如果元素存在则返回下标
			如果元素不存在就返回本来应该插入位置的（负下标）-1，
			例{1,2,3,4}查找5
					5本来应该插在下标为4的位置
					所以为-4-1=-5；
## 2.ArrayList
***底层原理：***
	利用空参构造创建时，默认长度为0
	添加第一个元素后，会创建一个长度为10的***新数组***
	存满后会扩容1.5倍，（创建新数组，在迁移元素）
	如果一次添加元素就超过了1.5倍`addall(list)`则新数组长度以实际元素个数为准
	
	ArrayList<Integer> list;
	增：
			list.add(元素)->默认为尾插
			list.add(下标,元素)
	删：
			list.remove(e)
			注：如果e为int类型，则按下标删除，反之如果和list元素数据类型一样，则按值删除,常数为int
	改：
			 list.set(下标，新的元素)，返回被修改元素
	查：
			 list.size()->返回元素个数
			 list.get(下标)
	forEach(lambda)
	例:list.forEach(i -> System.out.println(i));
## 3.LinkedList(双向链表)

**注：底层中，头尾节点不是开始就有的，开始均为空，如果添加了一个元素，则这个元素，既是头节点也是尾节点**

	LinkedList<Integer> list;
	增：
			list.addFirst(元素)->头插法
			list.addLast(元素)->尾插法
			list.add(元素)->默认为尾插
			list.add(下标,元素)
	删：
			list.removeFirst()->删除并返回第一个元素
			list.removeLast()->删除并返回最后一个元素
			list.remove(e)
			注：如果e为int类型，则按下标删除，反之如果和list元素数据类型一样，则按值删除,常数为int
	改：
			 list.set(下标，新的元素)，返回被修改元素
	查：
			 list.size()->返回元素个数
			 list.get(下标)
			 list.getFirst();
			 list.getLast();
	forEach(lambda)
	例:list.forEach(i -> System.out.println(i));
## 4.HashSet

**原理：（哈希表）会默认创建(添加元素后）一个长度为16的数组，加载因子为0.75，数组名为table,当存入16\**0.75后就会扩容为原来的两倍**

**去重原理：如果根据哈希值算出来的地方，已经有元素了，则调用equals，和她比较属性值，如果相同则不再添加，反之添加，这就是为什么，自定义类型要重写hashCode(object的是根据地址计算的，而不是属性),equals（默认的时判断地址）**

**JDK8以前用的数组加链表（拉链法）
JDK8以后用的数组加链表加红黑树，（当链表长度超过8，且数组长度大于等于64，链表自动转化为红黑树**

***注：当存入为自定义数据类型时，必须重写hashCode和equals***

**特点：不重复，无序（是因为遍历方式不一样，而不是排序），无下标**

	HashSet<Integer> list;
	增：
			list.add(元素)
	删：
			list.remove(e)->按值删除
	查：
			 list.size()->返回元素个数
## 5.LinkedHashSet(继承hashSet)

**特点：有序，无索引，不重复**

**原理：哈希表，无红黑树，和hashSet相比，每个元素为记录其前面和后面的地址，形成双链表（尾插），遍历时从双链表头节点遍历，所以他是有序的

	HashSet<Integer> list;
	增：
			list.add(元素)
	删：
			list.remove(e)->按值删除
	查：
			 list.size()->返回元素个数
## 6.TreeSet

**特点：无重复，无索引，可排序(默认升序)**

**原理：红黑树**

**中序遍历输出**

自定义排序规则，通过重写，Comparable接口下的compareTo
**comparator优先级高于comparable

		fisrt()->第一个元素
		last()->最后一个元素

返回值：
	负数：认为要添加的元素是小的，再看左子树
	正数：存在右子树
	0:认为要添加的已经存在，合并
	同时，每一步都得依赖红黑树的规则

**一般都用1，当1解决不了时，用2**
	**自定义排序1：**
```java
public class Student implements Comparable<Student>{  
    private  int age;  
    private  String name;  
  
    @Override  
    public int compareTo(Student o) {  
        return this.age-o.age;  
    }  
}
```
**自定义排序2**
```java
TreeSet<String> set =new TreeSet<>((String o1,String o2)->{  
    if(o1.length()!=o2.length())return o1.length()-o2.length();  
    else return o1.charAt(0)-o2.charAt(0);  
});
```

## 1.HashMap

**特点：无序（是因为遍历方式不一样，而不是排序），不重复，无索引**

**原理：哈希表，根据键来确定哈希值，如果键一样（通过equals比较）则覆盖，其余和hashSet一样，也是挂在下面，超过一定数量则转化为红黑树**

	HashMap<String,Integer> map;
	增：
		Object value=map.put(Key,Value);//会返回Value
		如果该Key已经存在了，则会覆盖
	删：
		Object key=map.remove(Key);->按Key删除，并返回键
	改
		map.put(key,修改后的value);
	查：
		map.size();
		map.get(Key)->得到key对应的值
	其他：
		map.containsKey(key)->判断是否包含键
		map.containsValue(value)->判断是否包含值
		map.clear()->清空
		map.isEmpty();->判断是否为空
		Set<String> s=map.keySet();->得到所有键，并以set的形式返回键，可结合map.get()，遍历map
		Set<Map.Entry<String,Integer>>s=map.entrySet();->得到所有键值对
